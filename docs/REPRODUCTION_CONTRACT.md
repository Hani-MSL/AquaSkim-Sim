# قرارداد بازتولید پروژه

## اجرای مرجع

فرمان نهایی انتشار پروژه، پس از تکمیل مراحل باقی‌مانده، بدون هیچ پرسش تعاملی اجرا خواهد شد:

```bat
scripts\build_reference_project.bat
```

این فرمان باید طراحی مرجع، همهٔ سناریوهای ثابت، تحلیل‌های محاسباتی، نمودارهای دو/سه‌بعدی، ویدئوها، Evidence، هش SHA-256، گزارش Word و ZIP تحویل را تولید کند.

## چرا Wizard حذف می‌شود؟

Wizard باعث می‌شود دو کاربر با پاسخ‌های متفاوت به خروجی‌های متفاوت برسند. این برای یک مخزن آموزشی ممکن است مفید باشد، اما برای «نسخهٔ رسمی قابل استناد گزارش» مناسب نیست. بنابراین:

- Build مرجع: بدون ورودی کاربر
- سناریوهای پژوهشی: فایل YAML نسخه‌دار
- پوشه‌های خروجی و Evidence: Git-ignored
- پارامترهای رسمی: Registry و Reference Design نسخه‌دار

## استفاده در GitHub

یک کاربر جدید باید فقط این مراحل را انجام دهد:

```bat
conda env create -f environment.yml
conda activate aquaskim-sim
python -m pip install -e .
scripts\build_reference_project.bat
```

تا زمان Freeze نهایی، اسکریپت Build مرجع فقط وضعیت انتشار را بررسی می‌کند و اجرای کامل نهایی را آغاز نمی‌کند.
